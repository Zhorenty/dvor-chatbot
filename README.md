# DVOR Telegram Bot

MVP-бот для спортивного объединения DVOR на Dart.

## Бизнес-документация

- Полная шпаргалка по пользовательским и админским сценариям, Google Sheets и операционным процессам: `BUSINESS_CHEATSHEET.md`

## Что умеет сейчас

- Личка (пользовательский сценарий):
  - `/start` — приветствие и постоянное кнопочное меню
  <!-- TODO(dvor-x-frank): вернуть кнопку `DVOR x FRANK by БАСТА` — промо-тизер спецпроекта (сверху меню) -->
  - `/trainings` или кнопка `Расписание` — сначала выбор категории (`Тренировки` / `Походы` / `Трейлы`), затем список по выбранной категории
    - в разделе `Тренировки` дата показывается лаконично с днем недели (например: `пн, 22.06.2026 19:00`)
  - `/book` или кнопка `Записаться` — сначала выбор категории, затем запись на конкретное мероприятие (только себя)
  - кнопка `Записать друга` — выбрать событие и ввести username/ФИО через запятую (до 5 человек), оплатить сумму `цена × число участников` одним чеком
  - после записи отправляет реквизиты/инструкцию оплаты
- `/profile` и кнопка `Профиль` — открывает профиль со статистикой и прогрессом по бонусам
  - внутри профиля: `Абонемент` — статус (`Normal`/`PRO`), условия PRO и кнопка `Оформить`
- `/my_bookings` (или кнопка `Мои записи` внутри профиля) — открывает список записей (актуальные/прошедшие), включая записи друзей/гостей, которыми вы управляете
  - для походов/трейлов в шаге оплаты можно выбрать формат: полная оплата или предоплата (аванс/задаток)
  - после оплаты пользователь отправляет в чат файл с подтверждением (документ/фото), после чего заявка уходит администратору
  - для записей в `pending_payment` автоматически отправляет напоминания об оплате
  - кнопка `Помощь` — краткая справка по доступным действиям
- Личка (админский сценарий), главное меню:
  - `Заявки на оплату` / `/payments_queue` — выбор категории, затем очередь оплат на проверку
  - `Управление записями`:
    - `Список записей` -> сегмент (`Активные`/`Архивные`) -> категория -> запись по страницам
    - для записи: редактирование (`оплата`, `username`, `мероприятие`), восстановление и мягкое удаление в архив
    - `Создать запись` -> категория -> мероприятие -> username -> статус -> подтверждение
    - после изменения бот спрашивает, нужно ли уведомить клиента
  - `Список записавшихся` — выбор категории, затем список участников
  - `Рассылка` — текст (HTML) и/или фото пользователям и/или в группу
  - `Инструменты`:
    - `Обновить Google Sheets` — синхронизация расписания, тренеров и промокодов
    - `Экономическая сводка` / `/economic_summary <period>`
    - `Абонементы` — фильтры (`Активные` / `Скоро истекают` / `На проверке` / `Отмененные`) и `Поиск`; `На проверке` — очередь модерации
    - `Список дворян`, `Поиск по пользователю`
    - `Клиентское меню` — обычное пользовательское меню с кнопкой `Админ-меню` для возврата
  - `/approve_payment <id>`, `/approve_partial_payment <id>`, `/reject_payment <id>`
  - `/approve_subscription <id>`, `/reject_subscription <id>`, `/cancel_subscription <id>`
  - после модерации бот уведомляет пользователя; при `ADMIN_CHAT_ID` дублирует результат в админ-чат
  - при `ADMIN_CHAT_ID` автоматически шлёт экономическую сводку за завершённую неделю/месяц
- Групповой сценарий:
  - при входе нового участника бот отправляет welcome-сообщение в группу
  - welcome автоматически удаляется позже (или после `/start` в личке)
  - бот публикует анонс тренировки по правилам времени:
    - если тренировка в `16:00` или позже — в `12:00` в день тренировки
    - если тренировка раньше `16:00` — в `20:00` за день до тренировки
  - регулярная рассылка расписания в группу (`TARGET_CHAT_ID`, таймзона `TIMEZONE_OFFSET_HOURS`):
    - воскресенье `18:30`
    - вторник `10:00`
    - четверг `10:00`
  - рассылка реферальной программы в группу: среда `10:00`
  - антиспам в целевой группе: удаляет рекламу заработка/удалёнки с призывом «в лс» и банит отправителя
    - админы из `ADMIN_USER_IDS` не проверяются
    - при заданном `ADMIN_CHAT_ID` бот шлёт короткое уведомление о срабатывании
    - выключается через `ANTISPAM_ENABLED=false`
- Источники расписания:
  - `static` (по умолчанию) — расписание в коде
  - `google_sheets` — расписание из Google Sheets CSV с периодическим sync и fallback на последний валидный кэш

## Структура проекта

- `bin/dvor_bot.dart` — entrypoint и запуск бота
- `lib/src/config/app_config.dart` — конфигурация из CLI/env/.env
- `lib/src/telegram/telegram_client.dart` — Telegram Bot API клиент
- `lib/src/bot/handlers/private_handlers.dart` — логика private-сценариев и кнопочного меню
- `lib/src/bot/handlers/group_handlers.dart` — welcome flow и антиспам в группе
- `lib/src/application/group_spam_detector.dart` — эвристики рекламы заработка / удалёнки
- `lib/src/data/static_schedule_repository.dart` — статичное расписание тренировок
- `lib/src/data/google_sheets_schedule_repository.dart` — расписание из Google Sheets CSV
- `lib/src/data/sqlite_booking_repository.dart` — SQLite-хранилище записей и оплат
- `lib/src/messages/message_templates.dart` — шаблоны сообщений
- `test/` — unit-тесты

## Конфигурация

Создайте `.env` рядом с `pubspec.yaml`:

```env
BOT_TOKEN=123456:ABCDEF
TARGET_CHAT_ID=-1001234567890
SEND_GROUP_FALLBACK=true
POLL_TIMEOUT_SECONDS=25
SCHEDULE_SOURCE=static
GOOGLE_SHEETS_CSV_URL=
SCHEDULE_SYNC_INTERVAL_SECONDS=300
ADMIN_USER_IDS=123456789
ADMIN_CHAT_ID=-1001234567890
BOOKINGS_DB_PATH=data/bookings.sqlite
PENDING_PAYMENT_TTL_MINUTES=30
TIMEZONE_OFFSET_HOURS=3
ANTISPAM_ENABLED=true
LOG_LEVEL=info
```

Можно задавать и через CLI (имеет более высокий приоритет):

```bash
dart run bin/dvor_bot.dart --token=123456:ABCDEF --target-chat-id=-1001234567890 --schedule-source=google_sheets --google-sheets-csv-url="https://docs.google.com/spreadsheets/d/.../export?format=csv&gid=0"
```

### Формат Google Sheets CSV

Тренировки (лист `gid=0`, URL задается через `GOOGLE_SHEETS_CSV_URL`), обязательные колонки:

- `title`
- `starts_at` (например `2026-06-05 19:00` или `05.06.2026 19:00`)
- `location`

Опциональные колонки:

- `participants_limit`
- `coach`
- `notes`
- `include_trainers_in_participants` (`1/0`, `true/false`, `yes/no`, `да/нет`) — учитывать ли тренеров из whitelist как участников для лимита мест

Whitelist тренеров для автозаписи без шага подтверждения оплаты задается в `lib/src/config/trainer_booking_whitelist.dart`.

Список тренеров для кнопки `Тренерский штаб` загружается из отдельного листа (`gid=195037978`).
Используются колонки:

- `name` (обязательно)
- `link` (обязательно)
- `description` (обязательно)
- `role` (опционально, роль/направление тренера в кратком списке)

Походы и трейлы загружаются из тех же Google Sheets автоматически:

- Походы: `gid=294119056`
- Трейлы: `gid=1220729038`

Для них используются колонки:

- `title` (обязательно)
- `date_from` (обязательно)
- `date_to` (опционально, для однодневных событий можно оставить пустым)
- `description` (обязательно)
- `price` (опционально)

Промокоды загружаются из отдельного листа (`gid=432112868`) той же таблицы.
Используются колонки (заголовки нормализуются: lowercase, пробелы → `_`, поддерживаются алиасы):

- `code` (алиасы: `promocode`, `промокод`, `код`) — текст промокода, обязательно
- `discount_percent` (алиасы: `discount`, `percent`, `скидка`, `процент`) — целое число 1–100, знак `%` допускается; обязательно
- `categories` (алиасы: `category`, `категория`, `категории`, `тип`, `тип_мероприятия`, `мероприятия`) — список категорий через запятую или `;`; пустое значение либо `все`/`all` означает, что промокод действует на все категории

Строки без `code` или с невалидным `discount_percent` пропускаются. При дублировании кода побеждает последняя строка.
Промокод можно ввести на шаге подтверждения оплаты (кнопка `🎟 Ввести промокод`): при скидке меньше 100% пересчитывается сумма к оплате, при 100% запись становится бесплатной без подтверждения оплаты.

## Локальный запуск

1. Установите Dart SDK (3.5+).
2. Установите зависимости:
   - `dart pub get`
3. Запустите бота:
   - `make bot` (или `dart run bin/dvor_bot.dart`)
4. Быстрые команды через `Makefile`:
   - `make bot` — запуск бота
   - `make bot-stop` — остановка запущенного процесса бота
   - `make bot-restart` — перезапуск одной командой

## Проверка работоспособности (smoke test)

1. Откройте личку с ботом, нажмите `Start`.
2. Нажмите `Расписание` и `Помощь` — проверьте ответы.
3. Откройте `Профиль` → `Абонемент`, оформите заявку и отправьте файл с подтверждением (документ/фото).
4. Нажмите `/book` или `Записаться`, затем после оплаты отправьте файл с подтверждением (документ/фото).
5. Для записи компании нажмите `Записать друга`, добавьте участников и оплатите сумму за всех одним чеком.
6. Если ваш user id указан в `ADMIN_USER_IDS`, откройте `Инструменты` → `Обновить Google Sheets` и проверьте `Заявки на оплату` и `Абонементы` в инструментах.
6. Добавьте тестового пользователя в группу DVOR.
7. Убедитесь, что бот:
   - отправил welcome в группу,
   - удалил welcome после TTL или после `/start` в личке.

## Настройки в BotFather и группе

- Отключите privacy mode (`/setprivacy` → Disable): бот должен видеть обычные сообщения группы, иначе антиспам не сработает.
- Сделайте бота администратором группы с правами:
  - удаление сообщений,
  - блокировка пользователей.
- Убедитесь, что бот добавлен в нужную группу (`TARGET_CHAT_ID`).

## Деплой

### Вариант 1: Docker

1. Убедитесь, что в `.env` задан путь БД:
   - `BOOKINGS_DB_PATH=data/bookings.sqlite`
2. Создайте каталог для постоянного хранения SQLite:
   - `mkdir -p data`
3. Однократно (перед первым запуском с volume) перенесите текущую БД из контейнера, если она уже есть:
   - `docker cp dvor-chatbot:/app/data/bookings.sqlite data/bookings.sqlite 2>/dev/null || true`
4. Запустите сервис через Docker Compose:
   - `docker compose up -d --build`
5. `compose.yaml` уже монтирует `./data` в `/app/data`, поэтому БД сохраняется между `git pull`/пересборками контейнера.
6. Проверьте логи:
   - `docker logs -f dvor-chatbot`

### Вариант 2: systemd (Linux)

1. Скомпилируйте бинарник:
   - `dart compile exe bin/dvor_bot.dart -o dvor_bot.run`
2. Создайте unit-файл `dvor-bot.service` с `Restart=always`.
3. Передайте `EnvironmentFile=/path/to/.env`.
4. Запустите:
   - `sudo systemctl daemon-reload`
   - `sudo systemctl enable --now dvor-bot`
   - `sudo systemctl status dvor-bot`
